Uppgift

  • Ta med saker som kod-stöd (vilka programmerings-språk som går att använda), kostnad, för- och nackdelar och andra saker du tycker är utmärkande.
  • Beskriv även hur arbetssättet är för att publicera arbetsfiler till denna webbhost.

Jag har valt att publicera min webbsida via Github.

GitHub Pages är en statisk webbhostingtjänst som gör det möjligt att publicera webbplatser direkt från ett GitHub-repository. Den har stöd för följande typer av filer och teknologier:

  • HTML, CSS, JavaScript.
  • Markdown: GitHub Pages stöder även Markdown.
  • Jekyll integration
  • Nackdel är att eftersom GitHub Pages är en statisk host så stödjer den ingen backend-kod, som PHP, Python eller Node.js.

GitHub Pages är helt kostnadsfritt att använda för offentliga repositorys.

    Fördelar
  • Publicering sker automatiskt varje gång man pushar ändringar till sitt repository.
  • Ingen kostnad för att använda GitHub Pages.
  • Versionskontroll.
  • Stöd för anpassade domäner: Man kan använda en egen domän.
  • Integration med CI/CD: GitHub Pages kan integreras med GitHub Actions för automatiserade arbetsflöden som bygg, testning och publicering.
    Nackdelar
  • Begränsat till statiska webbplatser: GitHub Pages stöder endast statiskt innehåll, så server-side kod eller databaser kan inte användas.
  • GitHub Pages tillåter inte webbplatser större än 1 GB.
  • Github har en mjuk bandbreddsgräns på 100 GB per månad. För stora företag räcker detta inte till.
    Utmärkande egenskaper
  • Automatisk publicering när nya commits pushas till huvudgrenen. Ändringar publiceras automatiskt på webbplatsen.
Publiceringsprocess:
  • Skapa ett GitHub repository: Börja med att skapa ett repository på GitHub och ladda upp dina HTML-, CSS- och JavaScript-filer.
  • Aktivera GitHub Pages: Gå till "Settings" och sedan "Pages". Välj huvudgren och klicka på "Save". Nu kommer webbplatsen att publiceras på en URL som https://användarnamn.github.io/repo-namn/.
  • För att uppdatera webbplatsen, commita först ändringarna lokalt i koden och pusha sedan till repository. GitHub Pages uppdaterar webbplatsen automatiskt.
